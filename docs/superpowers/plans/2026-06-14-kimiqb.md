## Confirmed Inputs

- Official Kimi Skill docs require directory-form `SKILL.md` files to include explicit `name` and `description`; useful fields for KimiQB are `type: prompt`, `whenToUse`, `disableModelInvocation: false`, and `${KIMI_SKILL_DIR}` for bundled file references.
- Official Kimi Plugin docs support `kimi.plugin.json` or `.kimi-plugin/plugin.json`; this plan uses root `kimi.plugin.json` because it has precedence and matches the official datasource example.
- Kimi plugin `name` must match `[a-z0-9][a-z0-9_-]{0,63}`. Use internal id `kimiqb` and display name `KimiQB`.
- Kimi plugin install flow is `/plugins install <path-or-url>`, then `/reload` or `/new`; local installs are copied to `$KIMI_CODE_HOME/plugins/managed/<id>/`, so source edits require reinstall.
- Local shell check in this session returned `kimi: command not found`; release validation must not require the Kimi CLI, but docs must include Kimi CLI smoke steps for a machine where `kimi` is installed.
- CodexQB source is read-only. Current verified CodexQB status is clean on `main...origin/main`; keep it that way and do not depend on CodexQB binary assets for KimiQB v0.1.

    ("Use $codexqb", "/skill:kimiqb"),
    ("$codexqb", "/skill:kimiqb"),

    ("plugins/codexqb/skills/codexqb", "skills/kimiqb"),
    ("codexqb", "kimiqb"),
]


def read_source(relative: str) -> str:
    path = SOURCE_SKILL_ROOT / relative
    return path.read_text(encoding="utf-8")


def adapt_text(text: str) -> str:
    for old, new in TEXT_REPLACEMENTS:
        text = text.replace(old, new)
    return text


def write_text(relative: str, text: str) -> None:
    path = DEST_SKILL_ROOT / relative
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_text(text, encoding="utf-8")
